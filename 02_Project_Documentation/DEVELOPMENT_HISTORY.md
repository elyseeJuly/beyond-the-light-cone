# 宇宙群英传（LegendOfUni-rebuild）开发历程与里程碑文档

本项目致力于将经典的单机策略模拟游戏《宇宙群英传》（原 MFC/C++ 架构）重构并迁移至现代化的 Web 前端生态，以 React 19 + TypeScript + Vite 8 + TailwindCSS 4 构建高水准的沉浸式科幻模拟体验。

本篇文档记录了该项目从“MFC 遗产”走向“现代化 Web 模拟引擎”的完整开发历程、核心技术突破与未来展望。

---

## 📅 里程碑一：MFC 遗产迁移与 Web 基础构建（早期阶段）

### 1. 技术栈选型与升级
为了提供极致的图形流畅度与模块化的逻辑维护，项目放弃了陈旧的桌面框架，全面拥抱现代化 Web 栈：
- **核心框架**: React 19 (利用最新的并发渲染与自动批处理性能)
- **开发构建**: Vite 8 + TypeScript 5 (确保毫秒级热更新与严苛的静态类型类型防护)
- **视觉层级**: TailwindCSS 4 + Framer Motion (打造充满未来科幻感的磨砂玻璃态与动态微交互)

### 2. 静态数据架构重组
原版的核心规则库与数值参数被剥离并模块化为结构清晰的静态 JSON 资产，存储于 `src/data/` 下：
- `stars.json`: 恒星与行星的基础物理属性与容量数据
- `aliens.json`: 外星文明（三体、歌者、边缘世界等）的初始特质与 AI 个性类型
- `events.json`: 模拟过程中的各类历史转折事件与随机干扰项

## 📅 里程碑二：深度代码审计与致命缺陷诊断（2026年5月中旬）

随着自动化脚本与密集轮次模拟的介入，引擎中潜藏的系统级 Bug 和数值失衡被逐一暴漏。我们产出了全方位的审计报告：
- **`CODE_AUDIT_2026_05_18.md`**：记录了 12 个严重逻辑缺陷、7 个类型安全隐患及 5 个基础架构问题。
- **`STRATEGY_CURVE_ANALYSIS_2026_05_18.md`**：深入评估了游戏的数值曲线，并指出当时的整体策略平衡性仅有 **4/10**。

### 诊断出的致命设计缺陷：
1. **经济死亡陷阱 (Economy Deadlock)**：地球初始资源仅为 500 且没有基础经济支撑，导致玩家在 25 轮左右便会资源枯竭，直接陷入无法建设、无法殖民的死循环。
2. **智子封锁死锁 (Sophon Blockade Death-loop)**：智子封锁将基础物理研究速度强行惩罚缩减为 `/ 10`。这导致解锁对抗封锁的 `550W量子计算机` 需要长达 **900 个回合**，使得科研陷入不可逆的彻底瘫痪。
3. **威慑体系割裂 (Deterrence System Disconnection)**：面壁者积累的“威慑值”对三体等外星文明的进攻 AI **完全没有实际震慑作用**，AI 仅会检查执剑人个体的 `leadership` 属性。
4. **数字永生速通漏洞 (Digital Immortality Speedrun)**：数字方舟仅需 2 个前置科技、50 余回合即可研发完成，使得其他高难度胜利路径（如流浪地球、黑域计划）毫无存在价值。
5. **环境依赖崩溃 (Headless Context Crash)**：遗留的 URL 解析机制直接依赖于 Vite 专有的 `import.meta.env.BASE_URL`，导致所有无头测试脚本（Headless analyzer）在 Node.js 执行时直接抛出 `TypeError` 崩溃。

## 📅 里程碑三：核心重构与系统平衡性修复（2026年5月18日）

根据制定出的 `ITERATION_SPEC_V1.md` 优化规范，我们对核心代码发起了一次大范围的精准外科手术式重构，完全攻克了以上所有死穴。

### 1. 核心 Bug 的精准修复
- **工人分配动态化** (`EarthCivilization.ts`): 将原先写死的 `/ 90` 分母修正为动态的各部门比例总和，消除了玩家调整分配比例时发生的计算失真与人口蒸发。
- **军事舰队武装补全** (`Game.ts`): 补充了 `military` 事件添加舰队时缺少 Weapons 的逻辑，新舰队将正确附带主力武器“恒星级战舰”。
- **逃亡主义事件修正** (`Game.ts`): 纠正了 ADDTREACHERY 产生负面反转数值的错误，将其重命名为 `REDUCE_TREACHERY` 并与配置完美同步。
- **统一文化产出** (`EarthCivilization.ts`): 剔除了无用的遗留计算，将文化局的基础文化加成与员工产出科学合并，消除返回值被丢弃的隐患。

### 2. 星图重塑与外太空体系搭建
- **太阳系星图重归现实**: 补全了遗失的「海王星」，将所有行星的 Index 按物理位置重新排序，并重构了轨道缩放逻辑（内密外疏），大幅提升了拟真度。
- **无尽太空星图生成器**: 创建了 `StarGenerator.ts` 并与 `StarManager` 集成。利用基于种子的伪随机数发生器，为“50光年外”、“1万光年”、“整个银河系”自动生成了近 **900 个** 恒星及行星系，打通了深空探索与多级望远镜科技树的实体支撑。

### 3. 策略数值的全面平衡
- **科研回暖**: 智子封锁惩罚从 `/ 10` 缓和为 `/ 3`，解锁量子计算机的时间缩短至合理区间，避免玩家在中期陷入无限绝望。
- **威慑值起效**: 异星文明在决定发动黑暗森林打击或舰队入侵时，会将面壁者辛苦累积的 `deterrenceValue` 与执剑人的属性综合考量，使防卫对抗兼具策略深度。
- **科技树均衡**: 为数字永生胜利分支加入了「意识上传」等前置限制，封堵了 50 轮速通的缺陷；将“广播纪元”由转瞬即逝的 8 回合延长至 40 回合，给各种随机剧情和紧急选项留出了合理的消化空间。

## 📅 里程碑四：引入自动化集成测试与质量工程（2026年5月18日）

为确保任何后续接手项目的开发者或 AI 均能顺畅修改而不破坏核心功能，项目于今日正式引入了**现代化无头测试防御系统**。

### 1. Vitest 测试框架的全面集成
- 在 `03_Web_Rebuild/` 目录下完成 `vitest`、`jsdom` 与 `@testing-library/react` 等依赖项的安全装载。
- 配置了专属的 `src/test/setup.ts`，为 Node.js 无浏览器环境注入了全局的 `CustomEvent` 模拟器、`window.dispatchEvent` 及 `localStorage` 垫片，阻断了环境差异引起的执行报错。
- 引入了 `package.json` 的 `"test": "vitest run"` 自动化命令。

### 2. 核心测试防护网的建立
- 编写了 `src/test/core/Game.test.ts` 测试文件，覆盖了系统基础年份演进、危机纪元状态、地球初始人口状态以及事件标志位（Flag）的状态修改流程。
- 当前，通过执行单元测试可实现 **800ms 内对游戏基础模型逻辑的 100% 自动防退化验证**。

## 📅 里程碑五：新手教程系统全面重构（2026年5月18日）

原始教程仅包含 5 个纯文本引导页，内容过于概括，缺乏对界面布局、核心玩法机制、胜利路径和新手生存策略的具体指引，导致新玩家在进入游戏后无所适从。

### 重构前后对比

| 维度 | 重构前 | 重构后 |
|------|--------|--------|
| 教程步数 | 5 步 | **14 步** |
| 内容分类 | 无分类 | 5 大章节（序章/界面/玩法/胜利/生存指南） |
| 界面高亮 | 无 | 对应区域发光高亮（顶部/左侧/中央/右侧） |
| 操作提示 | 无 | 每步附带「小贴士」面板 |
| 键盘导航 | 无 | 方向键翻页 + Enter/Esc 快捷操作 |
| 进度可视化 | 简易圆点 | 进度条 + 章节标签 + 步骤圆点 + 步数计数 |
| 章节跳转 | 不支持 | 点击章节标签或圆点可自由跳转 |
| 返回上一步 | 不支持 | 支持 |
| 视觉效果 | 单色卡片 | 按章节主题渐变色彩 + 毛玻璃效果 |

### 14 步教程内容覆盖

1. **序章**（2步）：世界观引入 + 五大纪元时间线详解
2. **界面导览**（4步）：顶部资源面板、左侧导航、中央星图操作、右侧信息面板
3. **核心玩法**（4步）：经济运转原理、科技研发体系、面壁计划与威慑、外交与军事
4. **胜利之路**（1步）：六大胜利路径及其前置条件
5. **生存指南**（3步）：新手10步生存指南 + 常见失败原因 + 最终寄语

## 📅 里程碑六：深度交互优化与官方时间线、物理星图集成 (2026年5月18日)

本次迭代大幅增强了游戏的基础交互便利度、叙事连贯性，并引入了重量级的双轨时间线对比系统与真实物理太阳系轨道模型。

### 1. 基础交互体验与新手指导深度升级
- **随时唤起的新手教程**：在顶部系统操作栏（TopHUD）新增了“帮助 (?)”按钮，允许玩家在游玩过程中随时打开 14 步交互式新手教程。
- **革命性的教程挖孔（Clip Cutout）遮罩**：剔除了之前遮挡全屏的深色磨砂背景，重新设计了基于“反向投影挖孔 (Massive Box-Shadow Clip)”的 CSS 布局。现在当教程提示“左侧导航”、“右侧面板”或“顶部状态”时，教程对应的游戏界面会清晰无阻地展示在玩家眼前，实现了真正的情境教学。
- **智能指派排序算法**：彻底重构了 `PersonSelectPanel`。玩家在指派部门领导或面壁者时，系统会自动根据岗位的期望属性（例如面壁者看重统帅与艺术、天体物理学看重科研）将人才按能力评分从高到低自动降序排列，并为最适宜人才打上高亮推荐标签，极大地减少了决策摩擦。

### 2. 人物动态解锁与剧情锁机制
- 废除了开局即可调用所有后世名流（如罗辑、程心等）的逻辑。
- 修改了 `PersonManager` 初始配置，开局仅解锁处于“黄金岁月”初期奠基阶段的基础班底（如叶文洁、丁仪、常伟思等）。
- 升级了底层 `GameEventManager` 与 `events.json` 的 `effects` 系统。随着年份推进，罗辑、章北海、程心、维德等人物将伴随着对应的历史大事件（如面壁计划启动、增援未来计划、威慑纪元到来）解锁加入阵营。
- 修复了 `StoryModal` 弹出时由于全局 `opacity` 渲染错误导致的对话文本近乎全黑无法看清的致命 Bug，并将 onClose 回调标准化，彻底解决了点击“下一回合”可能发生的事件卡死。

### 3. 官方《三体》编年史与双轨对比系统
- **官方编年史**：整合官方设定的六大纪元（黄金、危机、威慑、广播、掩体、银河/黑域），数据化存储于 `timeline.json`。
- **游戏内观测面板**：左侧导航新增了“编年史观测 (Timeline)”专属视图组件 `TimelineViewer`，玩家可以实时比对当前年份在官方三体宇宙中的历史进度。
- **双轨对比系统**：重新设计了游戏结算画面 (`EndGameScreen`)。当游戏结束（无论是胜利还是毁灭）时，将展示 `TimelineComparisonPanel`，以双轨时间轴的形式并排陈列“原著时间线”与“玩家本局真实履历”，让玩家直观复盘每一次历史抉择的蝴蝶效应。

### 4. 真实物理太阳系星图重构 🪐
- **加入冥王星**：静态资源 `stars.json` 重载，冥王星（Index 10）正式列入太阳系天体中，提供专属的资源产出与人口上限，完成了太阳系全行星的完整集结。
- **二重卫星公转嵌套**：彻底改变了卫星绕太阳飞行的错误物理状态。月球被精密绑定在地球坐标系下，在星图运动中会丝滑地围绕着运动中的地球进行二重圆周公转。
- **开普勒第三定律与 Log 轨道**：应用开普勒第三定律引力公式（$v \propto r^{-1.5}$），令水星公转飞速，海王星与冥王星在外圈极其缓慢平稳地移动。轨道间距使用对数非线性处理，完美避免了行星间互相重叠或拉得过远而无法显示。
- **Hitbox 碰撞完全修复**：重构了 Canvas 视口拖动与缩放下的鼠标射击匹配算法，将屏幕鼠标坐标逆向映射为世界坐标。不论玩家如何进行大范围的 Zoom 与 Pan 拖拽，鼠标移动到任何行星上均能 100% 精确显示中文名称，点击 100% 呼出信息面板。
