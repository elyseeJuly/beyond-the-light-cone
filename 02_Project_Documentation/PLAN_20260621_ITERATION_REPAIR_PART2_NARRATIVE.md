# 迭代修复计划 - Part 2：叙事驱动与事件系统重塑
> **Date**: 2026-06-21  
> **Status**: Planned  
> **Category**: Iteration Plan (`PLAN_`)

本阶段聚焦于游戏核心的叙事节奏、纪元演进逻辑以及人物/文明状态机管理，解决因底层机制断层导致的叙事卡死和沉浸感破坏问题。

## 1. 风险与缺陷响应矩阵

| 风险/Bug | 问题描述 | 核心诉求 | 计划模块 |
|:---|:---|:---|:---|

## 2. 详细实施方案

### 2.1 叙事双轨合并：时间线演进优化 (P2)

**现状问题**：
游戏依赖“文化值”突破来演进纪元，但关键原著事件又被硬编码在“特定年份”触发。玩家在某年份未达到文化值，或提前达到文化值，都会导致剧情卡死。

**修复计划**：
1. **建立复合演进门槛 (Composite Milestone)**：
   - 纪元演进不再单一依赖文化值，而是结合“关键节点事件是否完成”和“年份阈值”共同判断。
   - 若年份到达但文化值不足，触发“文明停滞”类惩罚事件，而非卡死剧情；反之则触发“科技爆炸”类加速事件。
2. **事件触发池动态化**：
   - 将部分非绝对固定的“年份事件”转化为“前置条件满足后一定概率触发的阶段事件”，提升游玩的顺滑度。

### 2.2 纪元过渡与重大事件呈现 (Bug 2, Bug 7)

**现状问题**：
进入新纪元非常突兀，缺乏铺垫；重大事件面板没有与CG系统的深度结合，表现力薄弱。

**修复计划**：
1. **纪元过渡仪式感 (Bug 2)**：
   - 在文化值/年份满足纪元更迭条件时，不直接进入新纪元。
   - 强制插入一个“纪元末尾特殊大事件”（如：危机纪元末期的“面壁者计划启动”），玩家确认该事件后，伴随转场动画正式进入新纪元。
2. **重大事件CG联动 (Bug 7)**：
   - 重构重大事件弹窗组件（`MajorEventDialog.tsx`）。
   - 事件数据结构中新增 `cgRef` 字段，当触发重大事件时，直接在弹窗内播放对应的CG资源和文本描述。

### 2.3 文明与角色实体状态机修复 (Bug 6, Bug 9)

**现状问题**：
未发现的文明泄漏到 UI；已逝角色仍逗留在任命列表中，且死亡不发通知。

**修复计划**：
1. **文明发现控制 (Bug 6)**：
   - `CivilizationManager` 中的文明列表默认全部设为 `isDiscovered: false`。
   - 只有当特定探索事件（如“接收到三体舰队广播”）触发其 `onTrigger` 钩子，调用 `discoverCivilization(id)` 后，才允许在 UI 显示。
2. **角色生命周期管理 (Bug 9)**：
   - 角色模型增加 `birthYear` 和 `deathYear` 属性，或通过特定事件触发死亡。
   - 在 `GameLoop` 每回合末尾，检查角色存活状态：若已死亡，自动解除其所有职位，并向信息流（Information Stream / 弹窗）推送讣告。
   - 任命面板（`AppointmentPanel.tsx`）增加过滤器：仅显示 `isAlive === true` 的角色。
