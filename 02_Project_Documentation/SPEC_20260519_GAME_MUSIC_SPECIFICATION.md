## 一、音频系统架构设计

游戏已重构为**双音轨动态切换系统**，分别服务于两个核心阶段：

```
                    ┌────────────────────────┐
                    │      游戏主循环阶段      │
                    │   播放《岁月底座》纯音乐  │
                    │   BgmPlayer.tsx 控制器  │
                    └───────────┬────────────┘
                                │
                        [ 触发大结局条件 ]
                                │
                                ▼
                    ┌────────────────────────┐
                    │      大结局演出阶段      │
                    │  《岁月底座》淡出/暂停   │
                    │  《星屑》人声演唱渐入    │
                    │   CreditsRoll.tsx 播放  │
                    └────────────────────────┘
```

### 1.1 岁月底座 (years_base.mp3) — 正常游戏背景音乐
- **加载逻辑**：在玩家加载游戏时，由 TopHUD 的 `BgmPlayer` 异步初始化，加载 `/audio/years_base.mp3`。
- **循环机制**：设置 `audio.loop = true` 自动循环播放。
- **音量持久化**：音量及静音状态自动存入 `localStorage`，尊重玩家偏好并在刷新或重开局时继承。
- **自动淡出**：一旦捕获到 `game-over` 状态（无论是胜利还是失败分支），`BgmPlayer` 将立即暂停播放并重置状态，把音频通道干净地让出给终场主题曲。

### 1.2 星屑 (stardust.mp3) — 大结局人声主题曲
- **加载逻辑**：在 Phase 4（CreditsRoll）挂载时由终演模块异步初始化并淡入，加载 `/audio/stardust.mp3`。
- **循环机制**：`audio.loop = false` 播放单次，播放完毕后平滑过渡至静止状态。
- **动态指示器**：当音乐播放时，右上角将渲染基于 `lucide-react` 自适应的微动等离子柱状动态均衡器（Equalizer Bars），提升叙事演出的沉浸感与专业感。

## 三、音频状态控制与自愈机制

为了防止浏览器在未产生交互前阻止媒体自动播放（Autoplay Blocked），我们设计了多层容错防线：

1. **显式初始挂载**：静音（Muted）状态存储于本地。首次进入游戏默认以 `0.4` 音量启动，支持一键点击播放激活 AudioContext。
2. **交互引导**：`BgmPlayer` 包含精美的交互控制台（在顶栏右上角）。如果播放被浏览器策略阻止，UI 状态会恢复至“暂停”图标以供玩家手动点击激活。
3. **静默降级（Elegant Degradation）**：
   - 如果对应路径的音频文件尚未放入，系统控制台会友好地输出 `[BgmPlayer] BGM not found - running in silent mode` / `[CreditsRoll] Finale theme song not found`。
   - UI 层会自动将状态降级为精致的信息条：
     * 正常游戏时提示：`🔇 岁月底座 BGM 待加入`
     * 结局 Credits 时提示：`🎵 主题曲：《星屑》 待添加 → public/audio/stardust.mp3`
   - 此降级机制保证了**哪怕没有任何音频文件，游戏也能 100% 正常运行、交互、并渲染出绝美的结局特效与制作名单**。

## 四、美术资源进度报告

关于您的美术资源补充需求，当前的开发状态及说明如下：

### 4.1 自动分类头像自愈（已完成）
在 `GameEventManager.ts` 中，我们为对话系统新增了**基于关键词正则规则的 NPC 头像分类器**（`classifyAvatar()`）。当事件中使用 `default` 占位头像时，系统会根据发言人头衔（如：*“联合国发言人”*、*“安全局局长”*、*“破壁人”*）智能映射到对应的职业分类头像（例如 `npc_official.png`, `npc_military_commander.png`, `npc_rebel.png`）。

同时，我们在 `StoryModal.tsx` 中嵌入了 `onError` 自愈挂钩。**如果本地尚未放置生成的分类头像，浏览器加载失败时会无缝、无闪烁地降级回默认的水墨智子头像 `character_default.png`**，杜绝了页面中出现“裂开的图表/Broken Image”这种不专业的外观。

### 4.2 美术生成限制说明（已归档）
由于本轮会话中云端图片生成接口暂时触发了平台每小时的配额频控（429 Too Many Requests），本轮无法直接将图片文件写盘。

不过请不用担心！所有的 **9 张结局场景宽幅配图** 和 **10 张 NPC 职业分类头像** 的生成 Prompts 及技术规格我们已经整理并输出到了最详细的本地文档中：
👉 [SPEC_20260519_ENDING_ART_REQUIREMENTS.md](file:///Users/quantumrose/Documents/Emberois/LengendOfUni-rebuild/02_Project_Documentation/SPEC_20260519_ENDING_ART_REQUIREMENTS.md)

您可以直接根据此文档的 Prompt 将描述输入到 AI 生成工具中。生成完毕后，按照文档要求直接将图片放入 `public/images/` 即可。

### 4.3 已生成的资源状态与自愈方案
- **胜利/失败结局场景配图**：当前采用高保真三色混合渐变背景（根据不同结局氛围量身打造：如征服结局使用黑红橙金、黑域结局使用极简银墨），配合 9 类独立的画布粒子算法（如黑域的 `collapse` 塌陷星尘、数字永生的 `matrix` 赛博流雨、流浪地球的 `thrust` 引擎尾焰），即使没有配图也已具备惊艳的史诗演出效果。
- **NPC 头像**：如前述，直接自愈降级至 `character_default.png`。
