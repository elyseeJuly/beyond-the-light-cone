## 一、完全缺失的核心系统类（来自 SPEC_20260514_REBUILD_PART1 §8）

文档明确指定了 8 个新增类，**Web 重建版本中一个都没有独立实现**：

### 1.1 CWallfacer / 面壁者系统
- **文档规定**：独立 `Wallfacer.h/cpp` 类，包含秘密计划进度、叛变概率计算、INI 参数驱动
- **实际代码**：仅在 `EarthCivilization.ts` 中有 `wallfacers: Set<string>` 字段，每回合 `deterrenceValue += (leadership + art) * 0.5`
- **缺失**：
  - 无叛变判定逻辑（面壁者可能叛变，如希恩斯）
  - 无秘密计划进度追踪
  - 无 `wallfacer.ini` 配置文件
  - 无破壁人机制

### 1.2 CSwordholder / 执剑人机制
- **文档规定**：独立 `Swordholder.h/cpp` 类，包含威慑度维护、交接危机、冷却时间
- **实际代码**：仅在 `EarthCivilization.ts` 中有 `swordholder: string | null` 字段，`deterrenceValue` 简单累加
- **缺失**：
  - 无交接危机（执剑人更换时有叛变风险）
  - 无威慑度自然衰减
  - 无执剑人特殊属性影响

### 1.3 CDiplomacy / 外交系统
- **文档规定**：独立 `Diplomacy.h/cpp` 类，包含 0-100 好感度计分、6 级外交状态机、外交事件触发
- **实际代码**：`Game.ts` 中 `conductDiplomacy()` 方法仅有 4 种基础操作（negotiate/trade/provoke/alliance），使用 `FriendshipType` 枚举做简单加减
- **缺失**：
  - 无 6 级 `DiplomacyState` 状态机（虽然枚举已定义）
  - 无 `diplomacy.ini` 配置
  - 无外交事件触发
  - 无黑暗森林威慑下的外交特殊规则
  - 无多边外交/联盟体系

### 1.4 CEpochManager / 纪元管理器
- **文档规定**：独立 `EpochManager.h/cpp` 类，包含阈值检查、解锁内容通知、纪元切换事件
- **实际代码**：`Game.ts` 中 `updateEpoch()` 方法使用硬编码的年份范围（1-200 危机，201-260 威慑，261-300 广播，301-350 掩体，351+ 银河）
- **缺失**：
  - 无 `epoch.ini` 配置
  - 纪元切换仅基于年份，未基于文化值（与 PART1 中 `EPOCH_TYPE` 注释不符：文化0-199 危机，200-499 威慑等）
  - 无纪元解锁内容通知

### 1.5 CAIStrategy / AI 决策层
- **文档规定**：独立 `AIStrategy.h/cpp` 类，包含威胁评估、特殊武器使用判定、安全阀规则
- **实际代码**：AI 行为内联在 `AlienCivilization.ts` 的 `hunterBehavior/cleanerBehavior/expansionistBehavior/defensiveBehavior/opportunistBehavior` 五个方法中，逻辑高度简化
- **缺失**：
  - 无威胁评估公式（文档中的 `EvaluateThreat`）
  - 无水滴（Waterdrop）特殊武器使用
  - 无二向箔（DimensionStrike）特殊武器使用
  - 无引力波广播（GravityBroadcast）濒死广播
  - 安全阀总原则未实现（玩家仅剩1星不攻击、冷却时间、预警事件、军力阈值）

### 1.6 CThemeManager / 主题管理器
- **文档规定**：独立 `ThemeManager.h/cpp` 类，双主题色彩表管理
- **实际代码**：通过 TailwindCSS `dark:` 前缀 + CSS 变量实现，`App.tsx` 中直接切换 `dark` class
- **缺失**：
  - 无独立的主题配置类
  - 未实现文档指定的完整色彩表（全息青 `#00E5FF`、高能橙 `#FF5722` 等具体色值）

### 1.7 CPlanetEngine / 行星发动机子系统
- **文档规定**：独立 `PlanetEngine.h/cpp` 类，推进/变轨/流浪计划
- **实际代码**：完全缺失。行星发动机仅作为科技树节点存在（`行星发动机Ⅰ/Ⅱ/Ⅲ型`），无实际运行逻辑
- **缺失**：
  - 无发动机推进进度
  - 无变轨操作
  - 无月球危机应对
  - 无流浪地球计划执行

### 1.8 CDigitalLife / 数字生命系统
- **文档规定**：独立 `DigitalLife.h/cpp` 类，意识保存/复活/MOSS
- **实际代码**：完全缺失。数字生命仅作为科技树节点存在，无实际运行逻辑
- **缺失**：
  - 无意识上传机制
  - 无角色复活功能
  - 无 MOSS AI 自主防御
